Network of Thrones

SAND Edition

This map is a zoomable, searchable network visualization of the Storm of Swords Network. The data was published by Dr. Andrew Beveridge at Macalaster College and this visualization was done by Bobby Norton.

A. Beveridge and J. Shan, "Network of Thrones," Math Horizons Magazine , Vol. 23, No. 4 (2016), pp. 18-22.

The visualization leverages SAND, py2cytoscape, and python-igraph to implement the following features:

  • An edge's thickness represents its weight, i.e. the number of interactions between the characters.
  • The size of a vertex corresponds to its PageRank value.
  • The color of a vertex's border indicates its community.
  • The size of a vertex's label corresponds to its betweenness centrality.
  • The background of a vertex is a local copy of the character's profile image on http://gameofthrones.wikia.com.
  • The layout is a degree-sorted circle based on community, with saved and reloadable manual positioning of communities.

How did you make this?

You can find a Jupyter Notebook that will allow you to completely replicate this visualization on Github.

The app was made using Cytoscape.js based on their Wine and Cheese Demo.

Also see the SAND repo for examples of applying network analysis and visualization to systems architecture.

But...why?

SAND is a network analytics toolkit and workflow written in Python. SAND focuses on the domain of network data in systems architecture, but the principles are useful for network analysis in other domains such as this social network. The Network of Thrones dataset provided a basis to calibrate SAND against an existing well-described network analysis. Running this dataset through the SAND workflow started as a case study and functional test, but became useful and entertaining in its own right.
